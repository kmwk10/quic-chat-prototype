# QUIC Chat Prototype

Прототип мессенджера реального времени, построенный на базе протокола HTTP/3 и технологии WebTransport с использованием языка Go.

## Цель проекта

Демонстрация работы с WebTransport в Go. Проект реализует архитектуру с центральным компонентом Hub для управления состоянием подключений и рассылки сообщений.

## Структура проекта
```
/cmd
  /client     # Точка входа клиентского приложения
  /server     # Точка входа серверного приложения
/internal
  /chat       # Логика мессенджера (Hub, Client, Read/Write pumps)
  /server     # HTTP-хендлеры и инициализация WebTransport
```

## Запуск проекта

### 1. Подготовка
Клонируйте репозиторий и загрузите зависимости:

```bash
git clone https://github.com/kmwk10/quic-chat-prototype.git
cd quic-chat-prototype
go mod download
```

Для работы по протоколу HTTPS необходимы TLS-сертификаты. Вы можете сгенерировать их локально:\
**Linux / macOS**
```bash
go run $GOROOT/src/crypto/tls/generate_cert.go --host localhost
```

**Windows**
```PowerShell
go run "C:\Program Files\Go\src\crypto\tls\generate_cert.go" -host localhost
```

### 2. Запуск сервера
```bash
go run cmd/server/main.go
```

После запуска сервер будет слушать соединения на https://127.0.0.1:4433. 
Работа сервера будет логироваться в консоль.


### 3. Запуск клиента
В отдельном терминале запустите клиента:

```bash
go run cmd/client/main.go
```

Можно запустить несколько клиентов, чтобы проверить рассылку сообщений между ними.

## Клиент

Работа клиента строится в два этапа:

1. Проверка доступности (Health Check): Клиент выполняет GET-запрос на эндпоинт /health. Это подтверждает, что сервер запущен, протокол HTTP/3 доступен и готов к обработке соединений.
2. Установка соединения (WebTransport): После успешного ответа от /health клиент инициирует WebTransport-сессию для обмена сообщениями.

## Основные команды клиента

- Введите текст сообщения и нажмите Enter для отправки всем участникам чата.
- /exit - для выхода из чата.

## Архитектура

1. Hub: управляет списком активных клиентов через каналы Register/Unregister.
2. ReadPump: горутина клиента, слушающая входящие WebTransport стримы и передающая сообщения в Hub.
3. WritePump: горутина клиента, ожидающая сообщения в канале Send и отправляющая их в стрим.
